# 初始化当前目录为一个git仓库

Git工作目录下对于文件的修改（增加、删除、更新）会存在几个状态，这些修改的状态会随着我们执行Git的命令而发生变化

# 如何使用命令控制这些状态之间的转换
# 用命令新建一个txt
$ touch file01.txt

# 工作区->暂存区，表示把当前目录下的所有文件放在暂存区
$ git add .

# 使用查看暂存区命令可以看到file01.txt进入暂存区
$ git status
On branch master

No commits yet

Changes to be committed:
  (use "git rm --cached <file>..." to unstage)
        new file:   file01.txt

# 暂存区->本地仓库，提交所有暂存区到本地仓库，记得写本次操作别名
$ git commit -m "add file01"
[master (root-commit) 7bd0fa2] add file01
 1 file changed, 0 insertions(+), 0 deletions(-)
 create mode 100644 file01.txt
 
#再次使用status命令，发现暂存区clean
$ git status
On branch master
nothing to commit, working tree clean

# 使用log命令查看提交到哪去了
$ git log
commit 7bd0fa2a5688ec63999695d084afe9999d3dc6e8 (HEAD -> master)
Author: WHH <2319770566@qq.com>
Date:   Fri Sep 8 21:44:09 2023 +0800

    add file01

shell语法中：

[shift] + [insert] 表示粘贴 直接选中目标则表示复制 [ctrl] +[c] 表示结束

# 表示清除上面的命令
$ clear

# 表示以一种简洁的形式查看本地仓库内的提交版本
$ git log --pretty=oneline --abbrev-commit --all --graph

# 表示查看已经删除的记录
$ git reflog

# 表示把版本回退到对应的版本号，commitID表示版本号
$ git reset --hard commitID

$ touch .gitignore

# 如忽略以a为扩展名的文件
*.a

# 表示删除文件的命令，这里删除.gitignore
$ rm -rf .gitignore
$ rm .gitignore

# mv表示剪切文件，如果移动目标地址没有该文件则重命名，Linux经常拿来改名字
$ mv .gitingore .gitignore

# 分支

# 表示查看分支
$ git branch

# 表示新建一个分支dev01
$ git branch dev01

# 表示切换分支，这里切换到dev01
$ git checkout dev01

# 表示直接创建并切换分支
$ git checkout -b dev02

# 想把什么东西合并到某个分支，先切换到当前分支
# 表示把某分支合并到当前分支
$ git merga dev01

# 表示删除某分支，不能删除当前分支，-d表示删除时做各种检查，-D表示不做任何检查，直接删除
$ git branch -d dev02
$ git branch -D dev02

# 发生冲突
# 会提示冲突的文件在哪，直接点开粗暴修改你想要的，修改成希望合并的样子，然后添加到暂存区，提交到本地仓库
$ git add .
$ git commit

# 撤回上一次提交
$ git reset --hard

开发中分支使用原则与流程：

• master（生产）分支 线上开发，主分支，中小规模项目作为线上运行的应用对应的分支

• develop（开发）分支 是从master创建的分支，一般作为开发部门的主要开发分支，如果没有其他并行开发不同期上线要求，都可以在此版本进行开发，阶段开发完成后，需要是合并到master分支，准备上线

• feature/xxxx分支 从develop层创建的分支，一般是同期并行开发，但不同期上线时创建的分支，分支上的研发任务完成后合并到develop分支

• hotfix/xxxx分支 从master派生的分支，一般作为线上bug修复使用，修复完成后需要合并到master、test、develop分支。

• 还有一些其他分支，在此不再赘述，例如test分支（用于代码测试）、pre分支（预上线分支）等等

# 远程仓库——gitee

# 表示连接远程仓库，origin表示别名，后跟地址（复制）
$ git remote add origin git@gitee.com:wang-haohong/git-test01.git

# 查看远程仓库
$ git remote

# 表示把本地仓库推到远程仓库上，master可以换为其他分支
$ git push origin master

# 推送到远程仓库命令详解
# -f表示强制覆盖
# --set-upstream
$ git push [-f] [--set-upstream] [远端名称[本地分支名][:远端分支名]]

# 表示把本地的master推到远程仓库origin，--set-upstream推送到远端的同时并且建立起和远端分支的关联关系，下次直接push即可
$ git push --set-upstream origin master:master

# 表示查看更为详细的分支
$ git branch -vv

# 表示从远程仓库克隆代码到本地，后不加别名则以链接最后的文件名为名字
$ git clone git@gitee.com:wang-haohong/git-test01.git

# 抓取指令，把仓库的更新抓取到本地但不会进行合并
$ git fetch [remote name] [branch name]

# 合并远程分支到本地
$ git merge origin/master

# 拉取指令就是将远端仓库的修改拉到本地并自动进行合并，等同于fetch+merge，如果不指定远端名称和分支名，则抓取所有并更新当前分支
$ git pull [remote name] [branch name]
